The Coding Designer's

Survival Kit

A starter set of code and design tools
All bundled up and ready to rock

When you design web sites in the browser you're left alone in the dark without any of your familiar design tools. By nature there's no application you can load up and start building your design. Well, this isn't that. However I think it's the next best thing.

Instead of starting from square one everytime you begin a project I think we need a collection of good starting values in markup and css to start designing, and a library of javascript tools to help us rapidly prototype as we think creatively. In the Kit you'll find HTML inspired by HTML5 Boilerplate, a system for writing highly flexible and powerful css with Sass, Compass, Susy, and other frameworks, and a bevy of javascript libraries at your disposal.

Please check it out, and start using it to flex your creative muscles as you design your next site in the browser.

Go Get It

Break it Down

Much like the HTML5 Boilerplate does for html and front end development, The Coding Designer’s Survival Kit aims to collect the most useful best practices for designing in the browser. Over time I’ve collected and developed a number of good habits and tools for design thinking using the tools of front end development as creative tools. My aim with this project is to collect and document them all here and collaborate with other coding designers to make this better. I already use this project as starting point for all my work, and I’d like it to become that useful to others as well.

HTML

I’m a big fan of how the HTML5 Boilerplate is structured. It’s a well-documented collection of best practices, intended to get you off and running. I love it so much I’m including the whole thing here, and adopting the same philosophy. The Coding Designer’s Survival Kit is not meant to be upgraded. Download it, and follow the comments in the code to start adding all your own secret sauce in all the right places. The Kit is meant to help while staying out of the way.

Let Paul get you up to speed on H5BP.

CSS

I love CSS to pieces, but I never want to write another line of it now that I’ve switched to [Sass](http://sass-lang.com/). I’ve [written](http://birdsandmonkeys.com/blog/css-suckers-introduction-sass-compass) about this before, but the real trick in the Survival Kit is that it establishes an easy–to–follow workflow for writing clean and solid Sassified CSS. Let’s go through them.

  1. Compile (almost) all of your CSS into one http request
  2. Add a creative toolset in Sass without adding any unneeded css
  3. Use Compass to help make CSS3 and repetitive styling tasks simple and fun
  4. Use a CSS grid framework without resorting to non-semantic classes, or settling for a one–size–fits–all grid system
  5. Pick and choose a css reset
  6. It comes with Sass stubs for responsive @media queries, Modernizer fallbacks, and more
